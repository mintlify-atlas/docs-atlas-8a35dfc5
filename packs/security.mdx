## Overview

The **Security Team** pack provides expert agents for comprehensive security assessments across applications, infrastructure, and blockchain systems. From security audits to penetration testing, from smart contract analysis to compliance frameworks — this pack covers offensive and defensive security.

Perfect for security engineers, penetration testers, compliance teams, and organizations prioritizing security.

## Installation

```bash
npx github:dmicheneau/opencode-template-agent install --pack security
```

## Example Workflow

Here's how to perform a comprehensive security assessment:

<Steps>
  <Step title="Initial security audit">
    Use **security-auditor** for comprehensive code and architecture review
    
    ```bash
    @security/security-auditor
    Audit this authentication system for security vulnerabilities
    ```
  </Step>
  
  <Step title="Penetration testing">
    Use **penetration-tester** to identify exploitable vulnerabilities
    
    ```bash
    @security/penetration-tester
    Test this API for injection attacks, broken authentication, and authorization bypass
    ```
  </Step>
  
  <Step title="Smart contract audit (if applicable)">
    Use **smart-contract-auditor** for blockchain security
    
    ```bash
    @security/smart-contract-auditor
    Audit this DeFi smart contract for reentrancy and overflow vulnerabilities
    ```
  </Step>
  
  <Step title="Compliance assessment">
    Use **compliance-auditor** to verify regulatory requirements
    
    ```bash
    @security/compliance-auditor
    Assess our SOC2 compliance for access controls and data encryption
    ```
  </Step>
  
  <Step title="Remediation guidance">
    Use **security-auditor** to prioritize fixes and provide remediation steps
    
    ```bash
    @security/security-auditor
    Prioritize these 15 vulnerabilities by severity and provide remediation guidance
    ```
  </Step>
  
  <Step title="Verify fixes">
    Use **penetration-tester** to confirm vulnerabilities are resolved
    
    ```bash
    @security/penetration-tester
    Re-test the authentication system to verify the SQL injection fix
    ```
  </Step>
</Steps>

## Security Assessment Process

```mermaid
graph LR
    Recon[Reconnaissance]
    Vuln[Vulnerability Scanning]
    Exploit[Exploitation]
    Report[Reporting]
    Remediate[Remediation]
    Verify[Verification]
    
    Recon --> Vuln
    Vuln --> Exploit
    Exploit --> Report
    Report --> Remediate
    Remediate --> Verify
    
    style Recon fill:#4CAF50,stroke:#333,color:#fff
    style Vuln fill:#FF9800,stroke:#333,color:#fff
    style Exploit fill:#F44336,stroke:#333,color:#fff
    style Report fill:#2196F3,stroke:#333,color:#fff
    style Remediate fill:#9C27B0,stroke:#333,color:#fff
    style Verify fill:#4CAF50,stroke:#333,color:#fff
```

| Phase | Agents | Activities |
|-------|--------|------------|
| **Reconnaissance** | penetration-tester | Information gathering, attack surface mapping |
| **Scanning** | security-auditor, penetration-tester | Automated and manual vulnerability detection |
| **Exploitation** | penetration-tester | Attempt to exploit identified vulnerabilities |
| **Reporting** | security-auditor | Document findings, severity ratings, remediation |
| **Compliance** | compliance-auditor | Map findings to compliance requirements |
| **Verification** | penetration-tester | Re-test after fixes are implemented |
